# MR Construction Test FrontEnd

## Запуск приложения
npm install
npm run dev

## Подключение сервера
Для связи в БД и API запросов необходиом запустить сервер [MR-Construction-Test-Be](https://github.com/SunnatBakidjanov/MR-Construction-Test-Be)

## Краткое описание

### Данные и стейт
Для серверного стейта использован TanStack Query — он берёт на себя кэширование, загрузку и инвалидацию данных. Локальный UI-стейт управляется через useState без лишних стейт-менеджеров, так как глобального стейта в приложении нет.

### Формы
Формы реализованы через React Hook Form с кастомными хуками валидации. Ошибки от API пробрасываются в форму через setError.

### API
Все запросы инкапсулированы в api/ через Axios, типизированы через TypeScript-интерфейсы. Каждый запрос обёрнут в отдельный хук через useMutation или useQuery.

### Тема
Реализована через data-theme атрибут на html с CSS-переменными, состояние персистится в localStorage.

### UI
Адаптивный интерфейс на Tailwind CSS, loading-состояния на уровне каждого действия, ошибки отображаются inline в формах, на запросы на создание ресурсов приходят "тост" уведомлениия.
